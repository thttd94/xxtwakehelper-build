## Operational errors/lessons from event scan
- One async script failed with `download_text_file(OUT_REMOTE)` because the remote output file was not ready or endpoint returned HTTP 500; fix by waiting longer and limiting output size.
- Another async command failed printing scan text because Windows terminal encoding `cp1252` could not encode binary/UTF-8 characters. Fix by writing to file with UTF-8/errors ignore and printing only sanitized snippets, or set stdout encoding.
- XXTouch Lua environment is limited: commands like `find`, `which`, `sqlite3`, sometimes `ss` may not exist. Prefer Lua `file.list()` scanning and download DBs to Windows for Python SQLite analysis.
